Getting Started with KPG Run

Getting Started with KPG Run

This guide walks you through your first power system optimization using KPG Run, from launch to results analysis.

Prerequisites

Before starting, ensure you have:

  • ✅ KPG Run installed (Installation Guide →)
  • ✅ KPG193 test system data available
  • ✅ Julia environment configured with JuMP, Gurobi, Ipopt
  • ✅ Basic understanding of power systems (helpful but not required)

Launch KPG Run

  2. Parameter Configuration (center): Set test system, day, options

  4. Results Tab (right): Solution summary and details

Tutorial: Running DC-OPF

Let’s solve a DC Optimal Power Flow for the KPG193 system.

Step 1: Select Solver

  1. Click “DC-OPF” in the solver type selector (left panel)
  2. The interface updates to show DC-OPF-specific options
flowchart LR
    A[Solver Types] --> B[Economic Dispatch]
    A --> C[Unit Commitment]
    A --> D[DC-OPF ← Select this]
    A --> E[AC-OPF]
    
    style D fill:#e8f5e9

Step 2: Configure Parameters

Test System:

  • Select “KPG193 v1.2” from dropdown
  • System automatically loads network data

Day Selection:

  • Enter “200” (summer peak day, mid-July)
  • Profiles load for day 200

Hour Selection:

  • Enter “18” (evening peak, 6 PM)
  • Highest demand hour typically

Solver Options:

  • Leave default settings for first run
  • Time limit: 300 seconds
  • MIP gap: 0.01 (1%)

Step 3: Solve

  1. Click “Solve” button
  2. Watch console output in real-time:

Step 4: View Results

Click the “Results” tab to see:

Summary Statistics

Generator Dispatch

Locational Marginal Prices (LMPs)

Congested Lines

Step 5: Analyze Results

Key insights from this run:

  1. Capital region (Area 1) has high LMPs

    • Import-constrained due to line limits
    • Need local generation or transmission upgrades

  2. Jeju Island (Area 5) has highest LMPs

    • HVDC link is limiting factor
    • Island is expensive to serve

  3. Three critical transmission corridors

    • Lines at 99%+ utilization
    • Candidates for expansion

  4. Generation mix appropriate

    • Nuclear: 100% online (baseload)
    • Coal: 82.6% (mid-merit)
    • LNG: 51.9% (peaking for high demand)

Problem: “Slow solve”

Problem: “AC-OPF not converging”

  1. Try warm start from DC-OPF
  2. Relax voltage limits slightly (0.88-1.12 vs 0.9-1.1)
  3. Increase iteration limit
  4. Check for data errors (very high/low values)

Common Mistakes

❌ Mistake 1: Wrong Day Selection

Fix: Check day range (1-365)

❌ Mistake 2: Mismatched Test System

Fix: Use consistent version throughout

❌ Mistake 3: Unrealistic Parameters

Fix: Use realistic reserves (5-15%)

❌ Mistake 4: Ignoring Warnings

Action: Investigate warnings, may indicate:

  • Data issues
  • Binding constraints
  • System bottlenecks
