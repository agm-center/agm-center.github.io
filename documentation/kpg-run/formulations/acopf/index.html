Use AC-OPF when:

  • ✓ Validating dispatch feasibility
  • ✓ Voltage stability critical
  • ✓ Reactive power planning
  • ✓ Accurate loss calculation
  • ✓ Generator capability curves matter
  • ✓ Detailed operational studies

Use DC-OPF when:

  • Speed is critical
  • Voltage not a concern
  • Large-scale studies (many scenarios)
  • Market clearing (active power only)

Constraint Explanation

(1b) Active Power Balance

Kirchhoff’s Current Law (real part):

Physical meaning:

  • Generation minus load at bus
  • Equals net power flow out through transmission
  • Nonlinear in and

(1c) Reactive Power Balance

Kirchhoff’s Current Law (imaginary part):

Physical meaning:

  • Reactive generation minus reactive load
  • Equals net reactive flow through network
  • Controls voltage magnitude

Key insight: Reactive power is local (doesn’t travel far)

Voltage Control

Methods to control voltage:

  1. Generator excitation (increase Q output)
  2. Shunt capacitors (inject Q)
  3. Shunt reactors (absorb Q)
  4. Transformer taps (change voltage ratio)
  5. FACTS devices (dynamic Q injection)

Interior Point Method (Ipopt)

Most common solver for AC-OPF:

Algorithm:

  1. Barrier function: Convert inequality constraints to penalty
  2. Newton’s method: Solve KKT conditions iteratively
  3. Line search: Ensure progress at each step
  4. Convergence: Stop when KKT residuals small

Convergence Challenges

Why AC-OPF may not converge:

  1. Infeasible problem

    • Demand too high for capacity
    • Voltage constraints inconsistent
    • Reactive power insufficient

  2. Bad initialization

    • Start too far from solution
    • Solution: Use DC-OPF result as warm start

  3. Numerical issues

    • Ill-conditioned Hessian
    • Nearly binding constraints
    • Solution: Scale problem, adjust tolerances

  4. Local optima

    • May converge to suboptimal point
    • Solution: Try multiple starting points

Typical convergence rate for KPG193:

  • Success: 97% (v1.2+)
  • Iterations: 35-80
  • Time: 2-8 seconds
