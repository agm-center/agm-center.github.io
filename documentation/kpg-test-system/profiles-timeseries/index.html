Temporal Profiles

The KPG 193 test system provides comprehensive temporal data covering a full year of power system operation. This page explains the structure of hourly profile files and how to work with time-series data.

All temporal data is organized into 365 daily files, each containing 24 hourly values for all 193 buses.

Directory Structure

File naming convention:

  • Demand: daily_demand_[day].csv where day ∈ {1, 2, …, 365}
  • Renewables: renewables_[day].csv
  • Weather: weather_[day].csv

Demand Profiles

File format: daily_demand_[day].csv — Hourly active and reactive power demand for all 193 buses.

Column definitions:

ColumnDescriptionUnitRange
hourHour of the day1-24Hour 1 = 00:00-01:00, Hour 24 = 23:00-24:00
bus_idBus number1-193Matches bus numbering in network files
demandPActive power demandMWReal power consumption
demandQReactive power demandMVArTypically ~10% of active power

Data structure:

  • Rows per file: 24 hours × 193 buses = 4,632 rows
  • Total rows (all files): 4,632 × 365 = 1,690,680 rows
  • Reactive power: Fixed at approximately Q = 0.1 × P (power factor ~0.995)

Renewable Generation Profiles

File format: renewables_[day].csv — Hourly capacity factors for solar, wind, and hydro generation at all 193 buses.

Column definitions:

ColumnDescriptionUnitRange
hourHour of the day1-24Same as demand
bus_idBus number1-193Matches bus numbering
pv_profile_ratioSolar PV capacity factor0-1Fraction of installed capacity
wind_profile_ratioWind capacity factor0-1Fraction of installed capacity
hydro_profile_ratioHydro capacity factor0-1Fraction of installed capacity

How to use:

  • Rows per file: 24 hours × 193 buses = 4,632 rows
  • Capacity factors: Dimensionless ratios (0 = no generation, 1 = full capacity)
  • Actual generation: Multiply capacity factor by installed capacity from renewables_capacity/ files

Renewable profile characteristics:

Solar PV:

  • Night (hours 1-6, 19-24): Zero generation (no sunlight)
  • Dawn (hours 7-8): Ramp up from 0% to 30-50%
  • Midday (hours 11-14): Peak generation 80-95% capacity
  • Afternoon (hours 15-17): Gradual decline
  • Seasonal variation: Higher in summer (longer days, better sun angle)

Wind:

  • Temporal pattern: Less predictable, depends on weather
  • Typical: Higher at night and early morning
  • Geographic: Coastal and Jeju Island buses have higher factors
  • Seasonal: Higher in winter months (monsoon winds)

Hydro:

  • Base profile: Relatively steady, run-of-river dominant
  • Daily variation: Some correlation with demand patterns
  • Seasonal: Higher in summer (monsoon season rainfall)
  • Low variability: Most stable renewable source

Weather Data

File format: weather_[day].csv — Meteorological variables from Korea’s LDAPS (Local Data Assimilation and Prediction System) used to generate renewable profiles.

Column definitions:

ColumnDescriptionUnitPurpose
hourHour of the day1-24Time index
bus_idBus number1-193Location
net_downward_longwave_flux_W/m^2Longwave radiationW/m²Solar resource indicator
temperature_2m_KTemperature at 2m heightKelvinDemand modeling, panel efficiency
wind_u_93m_m/sWind velocity (east-west)m/sWind power calculation
wind_v_93m_m/sWind velocity (north-south)m/sWind power calculation
wind_speed_93m_m/sTotal wind speed at 93mm/sHub height wind speed

Data structure:

  • Rows per file: 24 hours × 193 buses = 4,632 rows
  • Source: Korea Meteorological Administration LDAPS dataset
  • Purpose: Generate renewable profiles and validate demand patterns
