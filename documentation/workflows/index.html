Workflow 1: Day-Ahead Market Simulation

Objective

Simulate day-ahead market clearing for a summer peak day, computing generator schedules, locational prices, and identifying transmission constraints.

Steps

sequenceDiagram
    participant Data as KPG Test System
    participant UC as KPG Run (UC)
    participant OPF as KPG Run (DC-OPF)
    participant View as KPG View
    
    Data->>UC: Load Day 200 profiles
    UC->>UC: Solve 24h commitment
    UC->>OPF: Pass commitment schedule
    
    loop Each Hour
        OPF->>OPF: Solve DC-OPF
        OPF->>OPF: Calculate LMPs
    end
    
    OPF->>View: Export GeoJSON
    View->>View: Visualize results

Step 1: Load Test System

Step 2: Solve Unit Commitment

In KPG Run GUI:

  1. Select “Unit Commitment”
  2. Set parameters:
    • Test System: KPG193 v1.5
    • Day: 200
    • Horizon: 24 hours
    • Reserves: 10% spinning
  3. Click “Solve”

Or via command line:

Results:

  • 24 hourly LMP profiles
  • Generator dispatch schedules
  • Congestion patterns
  • Total daily cost

Step 5: Convert to GeoJSON

Step 6: Visualize in KPG View

Launch KPG View:

In viewer:

  1. Click “Choose File”

  2. Select day200_h18.geojson

  3. Observe:

    • Capital region (red) has high LMPs
    • Southern region (green) has low LMPs
    • 3 transmission lines at 99%+ loading
    • Nuclear running at full capacity
    • LNG units providing flexibility

  4. Enable P-flow animation to see power transfer patterns

  5. Apply “Congested Lines” preset to highlight bottlenecks
