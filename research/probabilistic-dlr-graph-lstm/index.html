Probabilistic Dynamic Line Rating with Line Graph Convolutional LSTM

Dynamic Line Rating Probabilistic Forecasting Graph Neural Networks Power System Operation Renewable Integration

A probabilistic DLR forecasting model that captures extended spatio-temporal correlations across transmission networks using a Line Graph Convolutional LSTM, enabling safer and more economical grid operation.

By AGM Center Research Team

Overview

Dynamic Line Rating (DLR) allows transmission operators to unlock additional line capacity by adjusting line limits in real time based on ambient weather. However:

  • Deterministic DLR forecasts frequently misestimate true line limits
  • Operational risks (overloading, redispatch) increase
  • Lack of network-wide correlation modeling degrades accuracy
  • Integration into OPF or market operation remains poorly studied

This research proposes a network-wide probabilistic DLR forecasting model using a Line Graph Convolutional LSTM (LGCLSTM).
It simultaneously learns:

  • Spatial correlations across all transmission lines
  • Temporal patterns from multi-year historical data
  • Uncertainty distributions via quantile forecasting

Integrated with a day-ahead + real-time redispatch OPF, LGCLSTM significantly reduces operational cost, renewable curtailment, and risk—outperforming existing ML-based and physics-based baselines.

Motivation

1. Static line rating underutilizes existing infrastructure

SLR uses conservative assumptions (high temperature, low wind), leading to:

  • Chronic underuse of transmission capacity
  • Congestion during high renewable output
  • Unnecessary renewable curtailment

2. Deterministic DLR forecasting is insufficient

Forecast errors can cause:

  • Overestimated limits → thermal risk
  • Underestimated limits → inefficient dispatch

Fig. 1 (page 1) shows how deterministic forecasts miss true DLR trajectories.

3. Existing probabilistic DLR methods ignore network topology

Most prior works treat each line independently, missing:

  • Spatial wind correlations
  • Shared thermal behavior across corridors
  • Upstream/downstream flow influence

4. DLR must be evaluated in system operation context

Forecasting alone is not enough. The real metric is:

“Does the forecast reduce OPF/UC cost, congestion, redispatch, and curtailment?”

This work provides the first network-wide, probabilistic, and OPF-integrated evaluation.

Key Contributions

  1. LGCLSTM: A Line Graph Convolutional LSTM Architecture

    • Converts the grid into a line graph so each transmission line becomes a “node”
    • Uses multi-hop graph convolution to capture extended spatial interactions
    • Embeds LGCN directly into LSTM → joint spatio-temporal learning

  2. Probabilistic Forecasting via Quantile Learning

    • Outputs multiple quantiles (80%, 90%, 98%)
    • Enables operator-controlled risk for DLR limits

  3. Integration into Grid Operation

    • Day-ahead dispatch uses conservative DLR quantiles
    • Real-time redispatch uses true DLR
    • Full evaluation of operational cost, congestion, redispatch, and curtailment

  4. Extensive validation on TX-123BT (123-bus Texas backbone)

    • 5 years of weather + synthetic DLR data
    • Comparison with QRF, QLSTM, T-GCN, GCLSTM
    • LGCLSTM reduces total operational cost and renewable curtailment

LGCLSTM Architecture

Fig. 1 Overall framework

Line Graph Conversion

Original grid: buses = nodes, lines = edges →
Convert to a line graph, where:

  • Each transmission line becomes a node
  • Adjacent lines become neighbors
  • Ensures consistent feature dimension
  • Preserves line-to-line correlations

Multi-hop Line Graph Convolution

Instead of single-hop GCN aggregation:

[ A_L^k \rightarrow \bar{A}_L^k \rightarrow \hat{A}_L^k ]

  • Captures k-hop spatial dependencies
  • Avoids oversmoothing
  • Requires only 1 layer instead of stacking many GCNs

LGCLSTM Cell

LSTM gates modified to use graph-filtered features:

[ f_t = \sigma(\hat{A}L^k X{t-1} W_f + H_{t-1} U_f + b_f) ]

and similarly for ( i_t, o_t, g_t ).

Combines:

  • Graph message passing
  • Temporal memory
  • Bidirectional flow (forward & backward layers)

Quantile Layer

Each line outputs multiple quantiles:

[ \hat{y}^q_i = \psi^q_i([h^F_{i,T} || h^B_{i,T}]) ]

Loss: pinball / quantile loss.

Operational Impact

Day-Ahead Scheduling

  • LSTM underestimates risk → lowest DA cost but highest redispatch
  • QRF is too conservative → inefficient. Higher DA cost
  • LGCLSTM achieves best balance → lowest total cost

Real-Time Redispatch

LGCLSTM:

  • Requires far less upward redispatch than LSTM
  • Does not over-commit expensive generators like QRF
  • Achieves second-lowest redispatch, near QRF but with better DA schedules

Total Cost

Across all quantiles:

LGCLSTM yields the lowest total cost among forecasting models.

Risk (CVaR 10%)

  • LSTM = highest risk
  • SLR = worst overall
  • LGCLSTM minimizes CVaR at QL = 0.05 and 0.10

Curtailment

(Table VI)

  • SLR has highest curtailment (cannot exploit high-DLR hours)
  • LGCLSTM achieves lowest or second-lowest curtailment

Operational Conclusion

Probabilistic DLR → Better than perfect DLR(!)

Perfect DLR cannot absorb load forecast uncertainty, whereas probabilistic lower-quantile forecasts introduce safety margin that reduces extreme redispatch.

Impact

✔ Improved Transmission Utilization

Dynamic quantile-based limits boost flow while maintaining security.

✔ Lower Congestion & Redispatch

Network-aware forecasting prevents aggressive but fragile DA schedules.

✔ Enables Renewable Integration

DLR intervals reduce solar/wind curtailment relative to SLR.

✔ First Network-Wide DLR Forecasting Framework

Captures full-grid spatial correlations.

✔ Provides a Practical DLR → OPF Interface

Useful for:

  • DA-UC
  • Economic dispatch
  • Congestion forecasting
  • Market design under DLR uncertainty

Future Directions

  • Integration with Security-Constrained UC (SCUC)
  • Learning-based selection of optimal quantile levels
  • Joint forecasting of DLR & renewable generation
  • Incorporating real-time sensor-based DLR data
  • Deployment in operator RTD or market systems
